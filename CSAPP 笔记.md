# CSAPP 笔记

## 第一章 数据表示

### 1. 整数 Integer

==**无符号整数（Unsigned Integer）**==

最简单的情况：**没有正负号**

如果是 n 位二进制：
$$
范围：0 \sim 2^n - 1
$$
比如 8 位：

```c++
00000000 = 0
00000001 = 1
...
11111111 = 255
```

**==有符号整数（Signed Integer）==**

**有符号整数使用==补码==表示：加减法可以统一，硬件实现简单**

**正数的补码 = 原码 = 直接写二进制**

```c++
5 = 00000101
```

**负数的补码 = 反码（所有位取反）+ 1**

```c++
00000101 -> 11111010 -> 11111011
-5 = 11111011
```

**溢出问题：**

```c++
// 比如 8 位：
127 + 1
01111111
+00000001
-----------
10000000
// 结果变成：
10000000 = -128
// 这就是溢出
```



### 2. 浮点数 Float

**浮点数 = 科学计数法的二进制版本**

**浮点数在计算机里是：**
$$
(-1)^s \times 1.M \times 2^E
$$

- **s = 符号位**
- **M = 尾数（mantissa）**
- **E = 指数（exponent）**

==**单精度浮点数（32位）**==

| 符号位 | 指数位 | 尾数位 |
| ------ | ------ | ------ |
| 1 bit  | 8 bit  | 23 bit |

1️⃣ **符号位 s**

0 → 正
1 → 负

2️⃣ **指数 E（偏移表示）**
$$
E = E_{存储} - 127
$$
3️⃣ **尾数 M（隐藏的1）**

浮点数采用 **规格化表示**：
$$
1.M
$$
**真实尾数 = 1 + 小数部分**

4️⃣ **举个完整例子**

```c++
/* 表示 5.75
第一步：转二进制
	整数部分：5 = 101
	小数部分：0.75 = 0.11
	所以：5.75 = 101.11
第二步：规格化
	101.11 = 1.0111 × 2^2
第三步：填入三部分
	符号位：正数 → 0
	指数：真实指数 = 2，存储指数：2 + 127 = 129，129 的二进制：10000001
	尾数：去掉前面的 1，得 01110000000000000000000
最终：0 10000001 01110000000000000000000
*/

/*
单精度 float：
  约 ±10^38
  精度约 7 位十进制有效数字
双精度 double：
  约 ±10^308
  精度约 15~16 位十进制有效数字
*/

/*
为什么浮点数会有误差？
很多十进制小数无法用二进制精确表示，比如：0.1，在二进制里是无限循环的。
*/

/*
浮点数本质上是：
定点尾数+指数缩放，越大的数，间隔越大；越小的数，精度越高
*/
```

==**双精度浮点数（64 位）**==

| 符号位 | 指数位 | 尾数位 |
| ------ | ------ | ------ |
| 1 bit  | 11 bit | 52 bit |



### 3. 大端序与小端序

**假设我们有一个 32 位整数：**

```c++
0x12345678 // 一个16进制数 = 4个2进制位
```

**它占 4 个字节：**

```c++
12 34 56 78
```

**问题来了：这 4 个字节在内存中怎么排？**

**==大端序==**

**高位字节存放在低地址**
**低位字节存放在高地址**

```c++
地址:   0    1    2    3
内容:   12   34   56   78

```

**==小端序==**

**低位字节存放在低地址**
**高位字节存放在高地址**

```c++
地址:   0    1    2    3
内容:   78   56   34   12

```

**==怎么判断你的电脑是大端还是小端？==**

```c++
int x = 1;
char *p = (char*)&x;

if (*p == 1)
    printf("Little Endian");
else
    printf("Big Endian");

```

2️⃣**==操作数类型==**

对于：

```assembly
addq %rbx, %rax   # %rbx 是一个操作数，%rax 是一个操作数
```

**类型①：立即数**

就是常数，前面带 $ ，不占寄存器，不从内存读取

```assembly
movq $5, %rax
# 意思：
rax = 5
```

**类型②：寄存器**

CPU 内部，速度最快

```assembly
addq %rbx, %rax
# 意思：
rax = rax + rbx
```

**类型③：内存引用**

```assembly
movq (%rax), %rbx
# 意思：
rbx = 内存中地址为 rax 的 8 字节内容
```



3️⃣**==寻址方式==**

**类型①：普通寻址**

```assembly
movq 0x100, %rax  # rax = 内存地址 0x100 处的值

movq (%rax), %rbx # rbx = M[rax]

movq 8(%rax), %rbx # rbx = M[rax + 8]



```

**类型②：变址寻址**

它的标准形式是：

```assembly
D(base, index, scale) # 最终地址 = base + index*scale + D

movq 8(%rax, %rbx, 4), %rcx # rcx = M[rax + rbx*4 + 8]


```

在 x86 中，不能同时操作两个内存数

```assembly
addq (%rax), (%rbx) ❌


movq (%rax), %rcx ✅
addq %rcx, (%rbx)
```



4️⃣**==常用汇编指令==**

**①数据传送指令**

| 指令      | 格式          | 效果（数学表达）                  | 实际动作                                      |
| --------- | ------------- | --------------------------------- | --------------------------------------------- |
| **movq**  | **movq S, D** | **D ← S**                         | **将 8 字节（64 位）数据从源 S 复制到目的 D** |
| **pushq** | **pushq S**   | **rsp ← rsp - 8<br />M[rsp] ← S** | **栈顶指针下移 8 字节，把 S 写入栈顶**        |
| **popq**  | **popq D**    | **D ← M[rsp]<br />rsp ← rsp + 8** | **从栈顶取 8 字节到 D，然后栈顶上移**         |

| -------- | ------------- | ------------------- | ------------------------------------------------------------ |

| **inc**  | **inc D**     | **D ← D + 1**       | **加1**                                                      |

| not      | not D         | D ← ~D              | 按位取反                                                     |

| sub      | sub S, D      | D ← D - S           | 减                                                           |
| imul     | imul S, D     | D ← D × S           | 有符号乘                                                     |
| xor      | xor S, D      | D ← D ^ S           | 异或                                                         |
| or       | or S, D       | D ← D \| S          | 或                                                           |
| and      | and S, D      | D ← D & S           | 与                                                           |
| **sal**  | **sal k, D**  | **D ← D << k**      | **算术左移**                                                 |

**③转移控制**

| 指令       | 核心作用   | 对栈的影响                               | 对程序计数器 (PC/RIP) 的影响 | 等价理解              |
| ---------- | ---------- | ---------------------------------------- | ---------------------------- | --------------------- |
| **call Q** | 调用函数 Q | **rsp ← rsp - 8<br />M[rsp] ← 返回地址** | **PC ← Q 的入口地址**        | push 返回地址 + jmp Q |
| **ret**    | 从函数返回 | **取出栈顶地址<br />rsp ← rsp + 8**      | **PC ← 弹出的地址**          | pop 地址 + jmp 地址   |

返回地址为 call 指令后面那条指令的地址。

5️⃣**==条件代码==**

**在 x86-64 里，CPU 有一个特殊寄存器：**

> **RFLAGS（标志寄存器）**

| 标志   | 含义                            |
| ------ | ------------------------------- |
| **ZF** | **Zero Flag（结果是否为 0）**   |
| **SF** | **Sign Flag（结果是否为负）**   |
| **CF** | **Carry Flag（无符号溢出）**    |
| **OF** | **Overflow Flag（有符号溢出）** |

**当你执行算术运算时，比如：==addq、subq、cmpq==，CPU 会自动设置这些标志位。**

**例如：**

```assembly
subq %rbx, %rax
# 意思是：
rax = rax - rbx
/* 执行完后：
如果结果是 0 → ZF=1
如果结果是负数 → SF=1
如果发生无符号借位 → CF=1
如果发生有符号溢出 → OF=1
*/

cmp 指令是干嘛的？
cmpq %rbx, %rax # rax - rbx，但它不保存结果，只更新条件码
```

**根据条件码跳转，常见跳转指令：**

| 指令    | 含义                  |
| ------- | --------------------- |
| **je**  | **equal（ZF=1）**     |
| **jne** | **not equal**         |
| **jl**  | **less（有符号）**    |
| **jg**  | **greater（有符号）** |

**例子：**

```assembly
# if-else 结构
if (x > y)
    return x;
else
    return y;
可能变成：
cmpq %rsi, %rdi    # %rdi = x; %rsi = y
jle  L2
movq %rdi, %rax
ret
L2:
movq %rsi, %rax
ret
---------------------------------------------------------------------------------
# while 结构
while (x > 0) {
    x--;
}
可能变成：
L1:                 # 循环开始
cmpq $0, %rdi
jle L2              # 如果 x <= 0 跳出
subq $1, %rdi       # x--
jmp L1              # 回到开头
L2:                 # 循环结束
---------------------------------------------------------------------------------
# for 结构 (编译器会把它转成 while 形式。)
for (i = 0; i < 10; i++) {
    sum += i;
}
可能变成：
movl $0, %eax        # i = 0
L1:
cmpl $10, %eax
jge L2               # 如果 i >= 10 结束
addl %eax, %edx      # sum += i
addl $1, %eax        # i++
jmp L1
L2:

```

### 2. 过程调用

过程调用全过程：

1️⃣==**场景：主函数调用加法函数**==

假设 main 函数调用 add(10,20)。在 64 位下，前 6 个参数通过寄存器传递。

```properties
参数1(10) -> %rdi
参数2(20) -> %rsi
返回值 -> %rax
```

2️⃣**==汇编代码流程==**

```assembly
# --- Main 函数部分 ---
main:
    movq    $10, %rdi      # 第一个参数入寄存器
    movq    $20, %rsi      # 第二个参数入寄存器
    callq   add            # 1. 压入返回地址 2. 跳转到 add
    # ... 继续执行 ...

# --- Add 函数部分 ---
add:
    # [Prolog: 建立栈帧]
    pushq   %rbp           # 保存旧的栈底指针 (Caller's %rbp)
    movq    %rsp, %rbp     # 将当前栈顶设为新的栈底

    # [Body: 业务逻辑]
    movq    %rdi, %rax     # 把 a 放入返回值寄存器
    addq    %rsi, %rax     # %rax = a + b

    # [Epilog: 销毁栈帧]
    popq    %rbp           # 恢复旧的栈底，%rsp 自动上移
    retq                   # 弹出返回地址并跳回 main
```

3️⃣==**内存中栈的变化**==

来看从 **`call` **执行到 **`ret`** 返回，栈内存是如何演变的：

**==状态 ①==：执行 `callq` 之后，进入 `add` 之前**

`callq` 把 `main` 函数中的下一条指令地址压入栈顶。

```properties
地址 (Addr)      内存内容 (Value)           说明
0x7FF...F80    [  Main 内部数据  ]  <-- %rbp (Main 的栈底)
0x7FF...F78    [  返回地址 (RA)  ]  <-- %rsp (由 call 指令压入)
```

**==状态 ②==：执行完 `pushq %rbp` 和 `movq %rsp, %rbp` 后**

这是函数最稳定的时刻，`%rbp` 就像一个锚点。

```properties
地址 (Addr)      内存内容 (Value)           说明
0x7FF...F80    [  Main 内部数据  ]
0x7FF...F78    [  返回地址 (RA)  ]  <-- 位于 8(%rbp)
0x7FF...F70    [  旧的 %rbp (Old)]  <-- %rbp 和 %rsp 指向这里
```

**==状态 ③==：执行 `popq %rbp` 后，执行 `retq` 前**

准备撤离，`%rsp` 指向返回地址

```properties
地址 (Addr)      内存内容 (Value)           说明
0x7FF...F80    [  Main 内部数据  ]
0x7FF...F78    [  返回地址 (RA)  ]  <-- %rsp 指向这，准备被 ret 弹出
0x7FF...F70    [ (已释放的旧RBP) ]  <-- %rbp 已恢复到 0x7FF...F80
```

4️⃣**==栈帧的边界==：**

```properties
%rbp    -> 当前栈帧的底部。
0(%rbp) -> 被保存的旧栈指针。
8(%rbp) -> 返回地址。
16(%rbp)-> 函数第 7 个参数
```



### 3. 缓冲区溢出

计算机程序在运行时会为变量分配内存空间，称为**缓冲区（Buffer）**。如果程序在写入数据时没有检查数据长度，导致数据超过了预留空间，多出的数据就会覆盖掉物理内存中相邻的存储单元。

1️⃣**==示例代码==**

```c++
#include <stdio.h>

void vulnerable() {
    char buf[8];
    gets(buf);   // 非常危险
}

int main() {
    vulnerable();
    return 0;
}
```

2️⃣**==问题在哪里==**

`buf` 只有 8 字节。但是 `gets()`不检查输入长度。如果用户输入：

```assembly
AAAAAAAAAAAAAAAAAAAA
```

远远超过 8 字节。

3️⃣**==栈中布局==**

假设函数栈帧如下：

```properties
高地址
-----------------
返回地址
-----------------
旧 rbp
-----------------
buf[8]
-----------------
低地址

```

4️⃣**==当输入超过 8 字节时发生什么==**

假设输入 24 个字节：

```assembly
AAAAAAAAAAAAAAAAAAAAAAAA
```

写入顺序是：

```properties
先填满 buf[8]
然后继续往上写
覆盖旧 rbp
再继续写
覆盖返回地址
```

变成：

```properties
高地址
-----------------
AAAAAAAA   ← 返回地址被覆盖
-----------------
AAAAAAAA   ← 旧 rbp 被覆盖
-----------------
AAAAAAAA   ← buf
-----------------
低地址
```

5️⃣**==当函数执行 ret 时==**

ret 做什么？

```assembly
pop 返回地址
跳转
```

但现在“返回地址”已经被改成：

```assembly
0x4141414141414141   # A 的 ASCII 是 0x41
```

程序就会跳转到非法地址，结果：

```properties
Segmentation fault
```
