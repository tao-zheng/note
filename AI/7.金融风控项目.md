### 分箱（离散化）

- 概念：分箱就是将连续变量离散化，合并成较少的状态

- 分箱的作用

  - **分箱（离散化）后的特征对异常数据有很强的鲁棒性**
  - **单变量分箱为N个后，每个变量有单独的权重，相当于为模型引入了非线性，能够提升模型表达能力**

- 常用分箱方法：卡方分箱、决策树分箱、等频分箱、聚类分箱

  ![image-20200911165052210](D:/download/000%20%E4%BA%BA%E5%B7%A5%E6%99%BA%E8%83%BD%E9%85%8D%E5%A5%97%E8%AF%BE%E4%BB%B6/005%20%E9%87%91%E8%9E%8D%E9%A3%8E%E6%8E%A7%20-AI%E7%89%88/005%20%E9%87%91%E8%9E%8D%E9%A3%8E%E6%8E%A7-V5.0-AI%E7%89%88/day03/01-%E7%AC%94%E8%AE%B0/assets/bins.png)

  - 等距分箱：按数据的特征值的间距均匀切分，每个箱体的数值距离一样

    ![image-20200911165913606](D:/download/000%20%E4%BA%BA%E5%B7%A5%E6%99%BA%E8%83%BD%E9%85%8D%E5%A5%97%E8%AF%BE%E4%BB%B6/005%20%E9%87%91%E8%9E%8D%E9%A3%8E%E6%8E%A7%20-AI%E7%89%88/005%20%E9%87%91%E8%9E%8D%E9%A3%8E%E6%8E%A7-V5.0-AI%E7%89%88/day03/01-%E7%AC%94%E8%AE%B0/assets/bins2.png)

  - **卡方分箱**：使用卡方检验确定最优分箱阈值

    - 初始化：根据连续变量值大小进行排序，构建最初的离散化
    - 合并：**遍历相邻两项的卡方值，将卡方值最小的两组合并，不断重复直到满足分箱数目要求**

### 编码

- **类别特征**需要做编码处理，常见编码方法：Onehot Encoding、Label Encoding、WOE Encoding

  **Label Encoding（标签编码）**

  - **原理**：将每个类别映射为一个唯一的整数（如 “未婚”=0，“已婚”=1，“离异”=2）。
  - 特点：适用于**有明确顺序的类别特征**（如 “低 / 中 / 高”、“小学 / 中学 / 大学”）。

  **One-Hot Encoding（独热编码）**

  - **原理**：为每个类别创建一个二进制列（0 或 1），样本属于某类别则对应列记为 1，其他为 0。
  - 特点：适用于**无顺序的离散类别**，会增加特征维度（维度 = 类别数量）。

  **WOE（证据权重）**

  - `WOE = ln(该类别中目标=1的比例 / 该类别中目标=0的比例)`

  - 将分类变量转换为数值，数值不仅仅代表一个分类，还代表了这个分类的权重。

  | 婚姻状态 | Good | Bad  | G-B  | ln(G/B) | WOE    |
  | -------- | ---- | ---- | ---- | ------- | ------ |
  | 未婚     | 30%  | 20%  | 10%  | 0.405   | 0.405  |
  | 已婚     | 40%  | 10%  | 30%  | 1.386   | 1.386  |
  | 离异     | 10%  | 40%  | -30% | -1.386  | -1.386 |
  | 丧偶     | 20%  | 30%  | -10% | -0.405  | -0.405 |
  | 总计     | 100% | 100% |      |         |        |

## 3.可解释性

**能否准确梳理模型的决策依据。**如果某个模型可解释性好，则是说我们能通过某些方法理解模型结果产生的逻辑。

一般来说，算法越复杂，一般精度越好，可解释性越差。

按照可解释性的强弱关系，将算法分类如下：

- 第一梯度：线性回归、逻辑回归
- 第二梯度：集成学习（结果是多个树共同决定的）
- 第三梯度：支持向量机（把数据往高维空间映射，数据会失真）



# 特征筛选

## 1.toad库筛选

从几个角度衡量：区分度，相关性，稳定性

### 区分度

区分度是评估一个特征对好坏用户的区分性能的指标。刻画了一个特征对好坏用户分布的区分程度

- IV值越大，区分程度越大，IV值越小，区分程度越小

$$
\rm{IV = \sum_{k}(p^{k}_{good}-p^{k}_{bad})\ln{\frac{p^{k}_{good}}{p^{k}_{bad}}}}
$$

| 婚配 | good | bad  | p_good | p_bad | p_good-p_bad | ln(p_g/p_bad) | IV    |
| ---- | ---- | ---- | ------ | ----- | ------------ | ------------- | ----- |
| 未婚 | 40   | 30   | 50%    | 37.5% | 0.125        | 0.2877        | 0.036 |
| 已婚 | 30   | 40   | 37.5%  | 50%   | -0.125       | -0.2877       | 0.036 |
| 其他 | 10   | 10   | 12.5%  | 12.5% | 0            | 0             | 0     |
| 总计 | 80   | 80   | 100%   | 100%  | -            | -             | 0.072 |

- IV<0.02 区分度小  建模时不用 （xgboost,lightGMB 对IV值要求不高）
  IV [0.02,0.5] 区分度大   可以放到模型里
  IV > 0.5  单独取出作为一条规则使用，不参与模型训练

### 相关性

- **先计算特征列之间相关性，把相关性高的列去掉**
- **计算特征列和标签列之间相关性，把相关性低的列去掉**

可以使用toad库来过滤大量的特征，高缺失率、低iv和高度相关的特征一次性过滤掉

### 稳定性

特征稳定性评估两个时间段特征分布差异的度量

- 常用的特征稳定性的度量有Population Stability Index (群体稳定性指标，PSI)
- 当两个时间段的特征分布差异大，则PSI大，当两个时间段的特征分布差异小，则PSI小

$$
\rm{PSI = \sum_{k}(p^{k}_{actual}-p^{k}_{expect})\ln{\frac{p^{k}_{actual}}{p^{k}_{expect}}}}
$$

## 2.通用方法筛选

### Boruta

- Boruta算法是一种特征选择方法，使用特征的重要性来选取特征

- 原理

  ![image-20200911205322499](D:/download/000%20%E4%BA%BA%E5%B7%A5%E6%99%BA%E8%83%BD%E9%85%8D%E5%A5%97%E8%AF%BE%E4%BB%B6/005%20%E9%87%91%E8%9E%8D%E9%A3%8E%E6%8E%A7%20-AI%E7%89%88/005%20%E9%87%91%E8%9E%8D%E9%A3%8E%E6%8E%A7-V5.0-AI%E7%89%88/day04/01-%E7%AC%94%E8%AE%B0/assets/feature16.png)

  1.创建阴影特征: 对每个真实特征R，**随机打乱顺序**，得到阴影特征矩阵S，拼接到真实特征后面得到【R,S】

  2.用新的特征矩阵N作为输入，训练模型，得到真实特征和阴影特征的feature importances,

  3.取阴影特征feature importance的最大值S_max，真实特征中小于S_max的，被认为是不重要的特征

  4.删除不重要的特征，重复上述过程，直到满足条件


小结：Boruta算法会计算特征重要性。

### 方差膨胀系数（VIF）

- 方差膨胀系数 :衡量了一个特征被其他特征线性解释的程度

  - 如果一个特征是其他**一组特征的线性组合**，则不会在模型中提供额外的信息，可以去掉
  - 当VIF超过某个阈值的时候(5 < VIF < 10)可以考虑把这个$x_i$删除

- 案例：

  ```python
      VIF_list = [variance_inflation_factor(x,i) for i in range(x.shape[1])]
  ```

### 递归特征消除 (RFE)

- 使用排除法的方式训练模型，把模型性能下降最少的那个特征去掉，反复上述训练直到达到指定的特征个数

  `sklearn.feature_selection.RFE`

  ```python
  #如下的案例可以快速出结果
  from sklearn.svm import LinearSVC
  from sklearn.datasets import load_iris
  from sklearn.feature_selection import RFE
  X,y = load_iris(return_X_y=True)
  #分类模型，C是正则化参数，用于控制模型复杂度和泛化能力，默认是1.0
  lscv = LinearSVC(C=0.01)
  
  # n_features_to_select 要选几个特征,  step 一次删掉几个
  selector = RFE(lscv,n_features_to_select=2,step=1)
  selector.fit(X,y)
  
  # support_ 返回False的可以被删除, 返回True的留下
  selector.support_
  # 获取顺序 序号越大的是优先被删除的
  selector.ranking_
  ```

## 3.逻辑回归评分卡

### 评分映射方法

逻辑回归，结果是一个概率值，区间在（0，1）之间。把这个概率值，映射到一个比较大的区间上（350-950）分数上（芝麻信用分）。

- 举例:

  - $$score = 650+50 log_2(P_{正样本}/ P_{负样本})$$

    - $P_{正样本}$是样本非逾期的概率，$P_{负样本}$是样本逾期的概率

  - 逻辑回归评分卡如何与评分卡公式对应

    $$
    p = \frac{1}{1+e^{-t}} \\
    1-p = \frac{e^{-t}}{1+e^{-t}} \\
      ln(p/(1-p)) = ln(e^{t})=t
    $$


  $$ln(P_{正样本}/ P_{负样本}) = w_1x_1+w_2x_2+w_3x_3+... ...$$

- 由对数换底公式可知:

  $$log_2(P_{正样本}/ P_{负样本}) = ln(P_{正样本}/ P_{负样本}) /ln2 = w_1x_1+w_2x_2+w_3x_3+... /ln2$$

  只需要解出逻辑回归中每个特征的系数，然后将样本的每个特征值加权求和即可得到客户当前的信用评分

## 4.集成学习评分卡

### LightGBM

**lightGBM 主要基于GBDT优化，提升整体特性**

1. **高效的直方图算法**：LightGBM 将连续特征值离散化为直方图（Histogram），减少了计算分裂增益的复杂度.
2. **带深度限制的Leaf-wise的生长策略**：优先分裂增益最大的叶子节点，这种策略能更快收敛
3. **直接支持类别特征**：无需手动对类别特征进行 One-Hot 编码，LightGBM 会将类别特征转换为整数编码
4. **直接支持高效并行**：支持特征并行（不同机器处理不同特征）和数据并行（不同机器处理不同样本）

# 不均衡学习和异常点检测

## 1.样本不均衡简介

- 在风控场景下，负样本的占比要远远小于正样本的占比

- 样本不均衡解决方案
  - 代价敏感：通常对少数类样本进行**加权处理**，使得模型进行均衡训练
  - 采样算法：欠采样、**过采样**



### 代价敏感加权

- 代价敏感加权增大了负样本在模型中的贡献，但没有为模型引入新的信息，既没有解决选择偏误的问题，也没有带来负面影响。
- 逻辑回归通过参数**class_weight = 'balanced'** 调整正负样本的权重，可以使得正负样本总权重相同
- 使用之前逻辑回归评分卡的例子

### 过采样

- 过采样是常见的一种样本不均衡的解决方案，常用的过采样方法为**SMOTE-少数类别过采样技术**

#### SMOTE算法

- SMOTE算法基本步骤如下
  - **采样最邻近算法，计算出每个少数类样本的K个近邻**
  - **从K个近邻中随机挑选N个样本进行随机线性插值**
  - 构造新的少数类样本
  - 将新样本与原数据合成，产生新的训练集

#### SMOTE案例

- 由于SMOTE算法是基于样本空间进行插值的，会放大数据集中的噪声和异常，因此要对训练样本进行清洗。这里使用LightGBM算法对数据进行拟合，将预测结果较差的样本权重降低不参与SMOTE算法的插值过程。

## 2.异常点检测

### 异常点检测

异常点检测(Outlier detection)，是找出与预期对象的行为差异较大的对象的一个检测过程。

### 常用算法

#### LOF(Local Outlier Factor)

基于密度的LOF算法要更简单、直观，不需要对数据的分布做太多要求，还能量化每个数据点的异常程度。

LOF会为每一个数据点计算出一个分数，LOF ≈1 ⇒ 非异常      LOF ≫1 ⇒ 异常

**LOF相关概念**

- 首先要确定参数K，K是LOF计算时需要考虑的近邻点数量

- **K-邻近距离（k-distance）**：K确定下来，我们可以计算k-distance，即点到第k个邻居的距离。 

- **局部可达密度（local reachability density）**：lrd是k邻近距离平均值的倒数

- **LOF 局部异常因子（local outlier factor）**：某点的LOF= K个邻居的LRD的平均值/该点的LRD

案例：

```python
from pyod.models.lof import LOF
clf = LOF(n_neighbors=20, algorithm='auto') #n_neighbors K个最近的邻居 
clf.fit(x)
# out_pred就是异常点的概率值
train['out_pred'] = clf.predict_proba(x)[:,1]
```

#### IF(孤立森林)

**隔离树**：

- 每棵 iTree 是一种随机二叉树：从特征中随机选择一个维度，再在该维度的最大值和最小值之间随机选择一个划分点，递归划分数据，直到每个叶子节点只包含一个样本。算法通过构建多棵**隔离树（iTree）** 组成森林。
- 异常点由于更容易被孤立，在 iTree 中会更早到达叶子节点（路径长度更短）。

**异常分数计算**：

- 对每个样本，计算其在所有 iTree 中的平均路径长度（到达叶子节点的步数）。
- 路径长度越短，异常分数越高。分数范围为 [0,1]，接近 1 表示高度异常，接近 0 表示正常。

# 项目总体流程

1. **数据清洗（第一步）**
   - 处理缺失值（根据业务逻辑填充，如用 - 1 表示 “无记录”）。
   - 异常值检测与处理（LOF/IF 算法，或基于业务规则的剪枝，如收入 > 100 万视为异常）。
   - 去除重复样本、纠正数据错误（如日期格式、数值范围校验）。
2. **特征工程**
   - 特征衍生（基于业务逻辑生成新特征，如 “还款收入比 = 月供 / 月收入”，这是风控核心步骤，常比算法调优更重要）。
   - 类别特征编码（WOE 编码适用于逻辑回归，树模型可直接用标签编码）。
3. **特征筛选**
   - 先用 VIF 去除高共线性特征（针对逻辑回归）。
   - 再用 toad/boruta/RFE 筛选重要特征（结合业务意义，避免保留无业务解释的特征）。
4. **数据集划分**
   - 按时间顺序划分训练集、验证集、测试集（如用 2021-2022 年数据训练，2023 年数据验证），确保时序一致性。
5. **处理类别不平衡**
   - 仅对训练集使用 SMOTE 过采样（或树模型内置参数），验证集和测试集保持原始分布。
6. **模型训练与评估**
   - 逻辑回归：用于构建可解释的评分卡，重点关注 AUC、KS、PSI（稳定性）。
   - 树模型（LightGBM）：作为辅助模型验证效果，或用于复杂场景的预测。
7. **模型验证与上线**
   - 跨时间验证（确保不同时间段稳定性）、跨群体验证（如不同地区客户）。
   - 评分卡校准（将模型输出映射为 0-1000 分，确保分数分布符合业务预期）。

**风控场景的特殊注意事项**

1. **业务可解释性优先**：特征和模型必须符合风控逻辑（如 “收入越高违约率越低”），避免选用无业务意义的 “黑箱特征”。
2. **稳定性比预测能力更重要**：PSI（群体稳定性指标）需 < 0.2，确保模型在不同时间 / 人群上的表现稳定，否则易导致误拒或误批。
3. **避免数据泄露**：特征衍生时不能使用 “未来信息”（如用贷款发放后的行为预测是否违约），过采样、特征筛选必须基于训练集独立完成。
