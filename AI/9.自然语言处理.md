#### 2.3.1 One-Hot 词向量表示

```properties
针对每一个词汇，都会用一个向量表示，向量的长度是n,n代表去重之后的词汇总量
俗称：独热编码、01编码
```

One-Hot编码的缺点：割裂了词与词之间的联系、如果n过大，会导致占用大量的内存（维度爆炸）



#### 2.3.2 Word2Vec模型

```properties
word2vec是一种无监督的训练方法，本质是训练一个模型，将模型的参数矩阵当作所有词汇的词向量表示
两种训练方式：cbow(continuous bag of words)、skipgram
```

**CBOW**

```properties
给一段文本，选择一定的窗口，然后利用上下文预测中间目标词
```

- 实现过程：**每个one-hot编码的单词与变换矩阵**(即参数矩阵3x5, 这里的3是指最后得到的词向量维度)**相乘之后再相加,** 得到上下文表示矩阵(3x1)。接着, 将**上下文表示矩阵与变换矩阵**(参数矩阵5x3, 所有的变换矩阵共享参数)**相乘, 得到5x1的结果矩阵**, 它将与我们真正的目标矩阵的one-hot编码矩阵(5x1)进行损失的计算, 然后更新网络参数完成一次模型迭代.
- 最后窗口按序向后移动，重新更新参数，直到所有语料被遍历完成，得到最终的变换矩阵(3x5)，这个变换矩阵与每个词汇的one-hot编码(5x1)相乘，得到的3x1的矩阵就是该词汇的word2vec张量表示.

#### 2.3.3 WordEmbedding词向量

![1754299206922](C:\Users\17630\AppData\Roaming\Typora\typora-user-images\1754299206922.png)

- 定义：将词映射到指定维度的空间：词向量的一种表示方法

  ```python
  import jieba
  import torch.nn as nn
  import torch
  
  # 1. 准备并分词句子
  sentences = [
      '传智教育是一家上市公司，旗下有黑马程序员品牌。我是在黑马这里学习人工智能',
      "我爱自然语言处理"
  ]
  
  # 分词并获取不重复词汇
  word_list = []
  for s in sentences:
      word_list.extend(jieba.lcut(s))
  vocab = list(set(word_list))  # 去重
  vocab_size = len(vocab)
  
  # 2. 手动创建词汇-索引映射（替代Tokenizer）
  word_to_idx = {word: i for i, word in enumerate(vocab)}
  idx_to_word = {i: word for word, i in word_to_idx.items()}
  
  # 文本数值化（将词语转为索引）
  sentence2id = [word_to_idx[word] for word in word_list]
  
  # 3. 创建嵌入层并获取词向量
  embedding = nn.Embedding(vocab_size, 8)  # 词汇量×8维向量
  
  # 输出结果
  print('词汇列表:', vocab)
  print('数值化结果:', sentence2id)
  print('\n词向量:')
  for idx in range(vocab_size):
      word = idx_to_word[idx]
      vec = embedding.weight[idx,:].detach().numpy()
      print(f'{word:4} {vec}')
  
  ```

### 4.1 添加N-gram特征

```properties
给定一段文本序列, 其中n个词相邻共现特征即n-gram特征, 常用的n-gram特征是bi-gram和tri-gram特征, 分别对应n为2和3.

以句子 "我爱自然语言处理" 为例（按词分割）：
unigram（n=1）：["我", "爱", "自然语言", "处理"]
bigram（n=2）：["我 爱", "爱 自然语言", "自然语言 处理"]
trigram（n=3）：["我 爱 自然语言", "爱 自然语言 处理"]
```



### 4.2 文本长度规范及其作用

```properties
一般模型的输入需要等尺寸大小的矩阵, 因此在进入模型前需要对每条文本数值映射后的长度进行规范, 此时将根据句子长度分布分析出覆盖绝大多数文本的合理长度, 对超长文本进行截断,
```

```python
# 覆盖90%左右语料的最短长度（根据实际数据分布调整）
cutlen = 10

def padding(x_train):
    """
    对输入文本序列进行长度规范（截断或补齐）
    :param x_train: 文本的列表表示，形如: [[1, 32, 32, 61], [2, 54, 21, 7, 19]]
    :return: 长度统一为cutlen的序列列表
    """
    padded_sequences = []
    for seq in x_train:
        # 1. 截断：若序列长度超过cutlen，保留前cutlen个元素
        if len(seq) > cutlen:
            padded = seq[:cutlen]
        # 2. 补齐：若序列长度不足cutlen，用0在末尾补齐（0通常表示空值）
        else:
            padded = seq + [0] * (cutlen - len(seq))
        padded_sequences.append(padded)
    return padded_sequences

```

## 1. RNN模型

- 定义： 一般以序列数据为输入, 通过网络内部的结构设计有效捕捉序列之间的关系特征, 一般也是以序列形式进行输出。当前时间步的输出 = 当前时间步的输入+上一时间步的隐层输出。

**RNN模型的分类**

- 根据输入与输出结构
  - N vs N ：特点：输入和输出是等长的，范围：诗歌、对联
  - N vs 1：特点：输入不限定长度，输出是唯一的，范围：分类任务
  - 1 vs N：特点：输入是唯一的，输出不限定长度，范围：图片生成任务
  - N vs M（seq2seq架构）：特点：输入和输出不等长，范围：文本翻译任务等
- 根据RNN内部结构
  - 传统RNN、LSTM、Bi-LSTM、GRU、Bi-GRU

## 2. 传统RNN模型

- **每个神经元内部是如何计算的呢？**
  $$
  h_t = \tanh\left(W_{ih} x_t + b_{ih} + W_{hh} h_{(t-1)} + b_{hh}\right)
  $$
  ![1753788260642](C:\Users\17630\AppData\Roaming\Typora\typora-user-images\1753788260642.png)

  **1.保留上下文信息**
  RNN 通过隐藏状态存储历史信息，使当前输出不仅依赖于当前输入，还依赖于之前的序列内容。

  **2.实现简单，计算成本较低**
  相比 Transformer 等模型，基础 RNN 的参数量少，训练和推理时的计算资源需求较低

  **3.梯度消失 / 爆炸问题**
  RNN 在处理**长序列**时，反向传播过程中梯度会随着时间步增长而急剧衰减或放大，导致模型难以学习到长距离的依赖关系。

  **4.对短序列依赖更敏感**
  由于梯度消失问题，RNN 对**近期信息**的关注度远高于**远期信息**，即 “短期记忆” 较强，“长期记忆” 较弱。

  ### RNN 的输入、输出和隐藏状态

  假设我们使用 **`batch_first=True`**（推荐，更符合直观思维），定义以下参数：

  - `batch_size`：批次大小（一次输入的样本数）
  - `seq_len`：序列长度（每个样本包含的时间步数量）
  - `input_size`：输入特征维度（词嵌入后的维度）
  - `hidden_size`：RNN 隐藏层维度
  - `num_layers`：RNN 的层数
  - `vocab_size`：词汇表大小（输出类别数）

  **1. 输入 (`x`)**

  - **形状**：`(batch_size, seq_len)`
    这是经过词嵌入前的输入。经过`nn.Embedding`,形状变为：`(batch_size, seq_len, input_size)`

  **2. 隐藏状态 (`h`)**

  - **隐藏状态 (h0)**：`(num_layers, batch_size, hidden_size)`

  ###### 3. RNN 输出 (`out`)

  ```
  (batch_size, seq_len, hidden_size)
  ```

  - 后续需要通过全连接层（`nn.Linear`）映射到词汇表大小：`(batch_size, seq_len, vocab_size)`

```python
import torch 
import torch.nn as nn


rnn = nn.RNN(input_size=5, hidden_size=6, num_layers=1, batch_first=True)

# 改变输入数据的长度，这里 seq_len=4（对应句子 "what time is it" 的词汇数量假设为 4 ）
# 输入维度变为 [batch_size, seq_len, input_size] ，即 [3, 4, 5]
input = torch.randn(3, 4, 5)  

h0 = torch.randn(1, 3, 6)  

# 一次性输入文本，由于 batch_first=True ，rnn 会按 [batch_size, seq_len, input_size] 
output, hn = rnn(input, h0)
print(output.shape)
# output 形状为 [batch_size, seq_len, hidden_size] ，即 [3, 4, 6]
print(hn.shape)
# hn 形状为 [num_layers, batch_size, hidden_size] ，即[1, 3, 6],只与output的最后一个相同

## 逐个时间步（词）输入文本，演示动态处理过程
# 按 seq_len 维度拆分输入，每次取一个时间步的数据，形状变为 [batch_size, 1, input_size]
seq_len = input.shape[1]  
for idx in range(seq_len):
    tmp = input[:, idx, :].unsqueeze(1)  # 维度调整为 [3, 1, 5]
    output, h0 = rnn(tmp, h0)
    print(output.shape)
    # 每次 output 形状为 [batch_size, 1, hidden_size] ，即 [3, 1, 6]
    print(h0.shape)
    # h0 形状始终为 [num_layers, batch_size, hidden_size] ，即 [1, 3, 6]

```

  

## 3. LSTM模型

- **内部结构**：**遗忘门、输入门、细胞状态、输出门**
- **遗忘门部分结构图**：与传统RNN的内部结构计算非常相似,最后通过sigmoid函数进行激活得到f(t), 我们可以将f(t)看作是门值,  遗忘门门值将作用的上一层的细胞状态上, 代表遗忘过去的多少信息。![1754360693289](C:\Users\17630\AppData\Roaming\Typora\typora-user-images\1754360693289.png)

- BI-LSTM

  ```properties
  定义: Bi-LSTM即双向LSTM, 它没有改变LSTM本身任何的内部结构, 只是将LSTM应用两次且方向不同, 再将两次得到的LSTM结果进行拼接作为最终输出.这种结构能够捕捉语言语法中一些特定的前置或后置特征, 增强语义关联,但是模型参数和计算复杂度也随之增加了一倍
  ```

- LSTM优势:LSTM的门结构能够有效减缓长序列问题中可能出现的梯度消失或爆炸

- LSTM缺点:由于内部结构相对较复杂, 因此训练效率在同等算力下较传统RNN低很多

## 4. GRU模型

- GRU（Gated Recurrent Unit）也称门控循环单元结构, 它也是传统RNN的变体, 同LSTM一样能够有效捕捉长序列之间的语义关联, 缓解梯度消失或爆炸现象. 同时它的结构和计算要比LSTM更简单, 它的核心结构可以分为两个部分去解析：**更新门**、**重置门**
- GRU模型：首先计算**更新门和重置门的门值**, 分别是**z(t)和r(t)**，之后重置门门值作用在了h(t-1)上, **代表上一时间步传来的信息有多少可以被利用**。接着就是使用这个重置后的h(t-1)**进行基本的RNN计算**，得到新的h(t)。最后更新门的门值会作用在新的h(t)，而1-门值会作用在h(t-1)上, 随后将两者的结果相加, 得到最终的隐含状态输出h(t)。![1754364105407](C:\Users\17630\AppData\Roaming\Typora\typora-user-images\1754364105407.png)

-  **Bi-GRU介绍**：Bi-GRU与Bi-LSTM的逻辑相同, 都是不 改变其内部结构, 而是将模型应用两次且方向不同, 再将两次得到的LSTM结果进行拼接作为最终输出。
- GRU优点：相比LSTM，结构较为简单，能够和lstm一样缓解梯度消失问题		
- GRU缺点：RNN系列模型不能实现并行运算，数据量大的话，效率比较低

```python
因为: dataloader返回的结果x.shape = [batch_size, seq_len, input_size],因此我们的代码是batch_first=True，这样做的目的，可以直接承接x的输入。'''
import torch
import torch.nn as nn

class NameClassifierRNN(nn.Module):
    def __init__(self, input_size, hidden_size, num_classes, num_layers=1):
        super().__init__()
        # 模型核心参数
        self.input_size = input_size
        self.hidden_size = hidden_size
        self.num_classes = num_classes  # 修正原拼写错误 ouput_size → num_classes
        self.num_layers = num_layers 
        # RNN层（batch_first=True：输入形状为 [batch_size, seq_len, input_size]）
        self.rnn = nn.RNN(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True
        )
        # 全连接层：将RNN输出映射到类别空间
        self.fc = nn.Linear(hidden_size, num_classes)  # 更规范的命名：fc（全连接层）
        # 对数softmax层（用于分类概率输出，数值稳定性更好）
        # 若输入张量形状为[batch_size, num_classes]，dim=-1会在num_classes维度上计算softmax
        self.log_softmax = nn.LogSoftmax(dim=-1)

    def forward(self, x, hidden):
        # RNN前向传播：获取所有时间步输出和最终隐藏状态
        # rnn_out: [batch_size, seq_len, hidden_size]
        rnn_out, final_hidden = self.rnn(x, hidden)    
        # 提取最后一个时间步的输出（用于序列整体分类）
        # 支持任意batch_size，替代原代码中仅支持batch_size=1的硬编码
        last_step_out = rnn_out[:, -1, :]  # 形状：[batch_size, hidden_size]
        # 映射到类别空间并计算对数概率
        logits = self.fc(last_step_out)  # [batch_size, num_classes]
        pred_probs = self.log_softmax(logits)
        return pred_probs, final_hidden

    def init_hidden(self, batch_size=1):
        return torch.zeros(self.num_layers, batch_size, self.hidden_size)
```

## 1. 什么是注意力机制

### 1.1 Attention概念

```properties
“注意力机制”实际上就是想将人的感知方式、注意力的行为应用在机器上，让机器学会去感知数据中的重要和不重要的部分。
```

### 1.2 注意力机制分类

```properties
1、软注意力: 注意力权重值分布在0-1之间，关注所有的词汇，但是不同词汇根据权重大小关注的程度不一样。
2、硬注意力: 注意力权重值是0或者1，只关注哪些重要的部分，忽略次要的部分
3、自注意力: 通过输入项内部的"表决"来决定应该关注哪些输入项.
```



## 2. 注意力计算规则

计算规则的普遍性表达：

```properties
第一步: query和key进行相似度计算，得到attention_score
第二步: 对attention_score进行softmax归一化得到权重值，压缩数值到0-1之间
第三步: 对上一步的权重值和value进行融合，得到具有权重信息的新value
```



**举例：**

- **中文输入（编码器处理）**：“我 喜欢 编程”（3 个词）

- **编码器输出**：每个词的 3 维隐状态（作为注意力的**Key**和**Value**）：

  | 中文词 | Key (K)   | Value (V) |
  | ------ | --------- | --------- |
  | 我     | [1, 0, 2] | [1, 0, 2] |
  | 喜欢   | [3, 1, 0] | [3, 1, 0] |
  | 编程   | [0, 2, 1] | [0, 2, 1] |

- **解码器任务**：生成英文词 “I”“like”“programming”，每个步骤的**Query**来自解码器当前隐状态。

**步骤 1：生成第一个英文词 “I”**

- **解码器查询 \(Q_1\)**：初始隐状态（关注 “主语”）→ [2, 0, 1]

1. 计算 Query 与每个 Key 的相似度（点积）

点积公式：$\text{相似度} = Q \cdot K_i = q_1k_{i1} + q_2k_{i2} + q_3k_{i3}$

- 与 “我” 的相似度：\(2×1 + 0×0 + 1×2 = 2 + 0 + 2 = 4\)
- 与 “喜欢” 的相似度：\(2×3 + 0×1 + 1×0 = 6 + 0 + 0 = 6\)
- 与 “编程” 的相似度：\(2×0 + 0×2 + 1×1 = 0 + 0 + 1 = 1\) 相似度向量：\([4, 6, 1]\)

2. Softmax 归一化得到权重

- 指数计算：$\exp(4)≈54.6，\exp(6)≈403.4，\exp(1)≈2.7$
- 总和：\(54.6 + 403.4 + 2.7 ≈ 460.7\)
- 权重：\([54.6/460.7≈0.12, 403.4/460.7≈0.87, 2.7/460.7≈0.01]\)

3. 加权求和 Value 得到上下文向量 \(C_1\)

\(C_1 = 0.12×[1,0,2] + 0.87×[3,1,0] + 0.01×[0,2,1]\)

- 第 1 维：\(0.12×1 + 0.87×3 + 0.01×0 ≈ 0.12 + 2.61 + 0 = 2.73\)
- 第 2 维：\(0.12×0 + 0.87×1 + 0.01×2 ≈ 0 + 0.87 + 0.02 = 0.89\)
- 第 3 维：\(0.12×2 + 0.87×0 + 0.01×1 ≈ 0.24 + 0 + 0.01 = 0.25\)\(C_1 ≈ [2.73, 0.89, 0.25]\)（主要融合 “喜欢” 和 “我” 的信息，符合 “我→I” 的对应）

4. 生成结果：解码器结合 \(C_1\) 输出 “I”

**步骤 2：生成第二个英文词 “like”**

- **解码器查询 \(Q_2\)**：生成 “I” 后的隐状态（关注 “动词”）→ [0, 3, 2]

1. 相似度计算

- 与 “我”：\(0×1 + 3×0 + 2×2 = 0 + 0 + 4 = 4\)
- 与 “喜欢”：\(0×3 + 3×1 + 2×0 = 0 + 3 + 0 = 3\)
- 与 “编程”：\(0×0 + 3×2 + 2×1 = 0 + 6 + 2 = 8\) 相似度向量：\([4, 3, 8]\)

2. 权重计算（Softmax）

- 指数：$\exp(4)≈54.6，\exp(3)≈20.1，\exp(8)≈2981.0$
- 总和≈3055.7，权重：\([0.018, 0.007, 0.975]\)（“编程” 权重极高，因 “喜欢” 与 “编程” 关联紧密）

3. 上下文向量 \(C_2\)

\(C_2 ≈ 0.018×[1,0,2] + 0.007×[3,1,0] + 0.975×[0,2,1] ≈ [0.04, 1.95, 0.99]\)

4. 生成结果：结合 \(C_2\) 输出 “like”

**步骤 3：生成第三个英文词 “programming”**

- 类似逻辑，查询会更关注 “编程” 的 Key，最终生成对应英文词。

**核心结论**

上下文向量 C 是对中文输入的加权融合，维度与 Value 一致（3 维）。每个步骤的 C 会根据当前生成需求（如主语、动词）动态调整权重，让解码器 “聚焦” 于最相关的中文词，确保翻译准确性。



注意力计算方法：

将Q，K进行纵轴拼接, 做一次线性变化, 再使用softmax处理获得结果最后与V做张量乘法.
$$
\text{Attention}(Q, K, V) = \text{Softmax}\bigl(\text{Linear}([Q, K])\bigr) \cdot V
$$
将Q与K的转置做点积运算, 然后除以一个缩放系数, 再使用softmax处理获得结果最后与V做张量乘法.
$$
\text{Attention}(Q, K, V) = \text{Softmax}\Bigl(\frac{Q \cdot K^T}{\sqrt{d_k}}\Bigr) \cdot V
$$


注意力机制实现步骤（深度学习中）:

```properties
第一步: 按照注意力规则，对Q、K、V进行注意力的计算
第二步: 如果第一步是拼接操作，需要将Q和第一步计算的结果进行再次拼接，(类似残差神经网络)                      如果是点乘运算，Q和K、V相等,一般属于自注意力，不需要拼接
第三步: 我们需要将第二步的结果，进行线性变化，按照指定输出维度进行结果的表示
```

```python
# word_embedding作用：将文本词汇进行张量（向量）表示
# 为什么embedding之后要乘以根号下d_model
# 防止position encoding的信息覆盖我们的word embedding，所以进行一个数值增大

class Embeddings(nn.Module):
    def __init__(self, d_model, vocab):
        # 参数d_model 每个词汇的特征尺寸 词嵌入维度
        # 参数vocab   词汇表大小
        super(Embeddings, self).__init__()
        self.d_model = d_model
        self.vocab = vocab

        # 定义词嵌入层
        self.embed = nn.Embedding(self.vocab, self.d_model)

    def forward(self, x):
        # 将x传给self.lut并与根号下self.d_model相乘作为结果返回
        # x经过词嵌入后 增大x的值, 词嵌入后的embedding_vector+位置编码信息,值量纲差差不多
        return self.embed(x) * math.sqrt(self.d_model)
```

### 2.2 位置编码器的代码实现

```python
# 位置编码器作用：Transformer 编码器或解码器中缺乏位置信息，因此加入位置编码器，将词汇的位置可能代表的不同特征信息和 word_embedding 进行融合，以此来弥补位置信息的缺失。
# 为什么使用三角函数来进行位置编码：这种设计可以让模型更容易学习到相对位置关系，因为正弦和余弦函数的周期性特性使得不同位置之间的偏移可以通过三角函数的和角公式来表示。

class PositionalEncoding(nn.Module):
    def __init__(self, d_model, dropout, max_len=60):
        # 参数d_model 词嵌入维度 eg: 512个特征
        # 参数max_len 单词token个数 eg: 60个单词
        super(PositionalEncoding, self).__init__()

        # 定义dropout层
        self.dropout = nn.Dropout(p=dropout)

        # 思路：位置编码矩阵 + 特征矩阵 相当于给特征增加了位置信息
        # 定义位置编码矩阵PE eg pe[60, 512], 位置编码矩阵和特征矩阵形状是一样的
        pe = torch.zeros(max_len, d_model)

        # 定义位置列-矩阵position  数据形状[60,1] eg: [0,1,2,3,4...60]^T
        position = torch.arange(0, max_len).unsqueeze(1)

        # 定义变化矩阵div_term [1,256]
        # 计算公式对应原论文中的10000^(2i/d_model)的对数形式，避免数值溢出
        div_term = torch.exp(torch.arange(0,d_model, 2) * -(math.log(10000) / d_model))

        # 位置列-矩阵 @ 变化矩阵 做矩阵运算 [60*1]@ [1*256] ==> 60 *256
        # 矩阵相乘也就是行列对应位置相乘再相加，其含义，给每一个列属性（列特征）增加位置编码信息
        my_matmulres = position * div_term

        # # 偶数维度用正弦函数
        pe[:, 0::2] = torch.sin(my_matmulres)
        # # 奇数维度用余弦函数
        pe[:, 1::2] = torch.cos(my_matmulres)

        # 形状变化 [60,512]-->[1,60,512]
        pe = pe.unsqueeze(0)

        # 把pe位置编码矩阵注册成模型的持久缓冲区buffer;模型保存再加载时，可以和模型参数一样一同被加载
        # 什么是buffer: 对模型效果有帮助的，但是却不是模型结构中超参数或者参数，不参与模型训练
        self.register_buffer('pe', pe)

    def forward(self, x):
        # 注意：输入的x形状2*4*512  pe是1*60*512 形状 如何进行相加
        # 只需按照x的单词个数 给特征增加位置信息
        x = x + self.pe[:, :x.size(1), :].to(x.device, x.dtype)
        return self.dropout(x)

```

```python
# 掩码：掩就是遮掩、码就是张量。掩码是张量，掩码张量的作用是对另一个张量进行数据信息的掩盖。一般掩码张量是由0和1两种数字组成，至于是0对应位置或是1对应位置进行掩码，可以自己设定
# 掩码分类：
# Padding Mask: 句子补齐的PAD,去除影响---------->编码器
# Padding Mask 的作用就是：在计算注意力分数时，将<PAD>位置对应的分数设为极小值（如 - 1e9），经过 Softmax 后这些位置的注意力权重会趋近于 0，确保模型只关注真实的 token。

# Sequence Mask:防止未来信息被提前利用----------->解码器
# Sequence Mask 通过在注意力计算中屏蔽未来位置的分数（将其设为 -inf，使 Softmax 后概率趋近于 0），强制模型只能依赖 “过去” 的信息进行预测，保证训练逻辑的合理性。
def subsequent_mask(size):
    """
    用于Transformer解码器  
    Returns:形状为[1, size, size]的后续掩码张量，下三角为0，上三角为1，为1表示需要遮掩
    """
    # 直接使用PyTorch生成上三角矩阵，避免NumPy转换
    mask = torch.triu(torch.ones(1, size, size), diagonal=1)
    return mask
```

### 3.2 多头自注意力层

#### 3.2.1 计算规则

$$
\text{Attention}(Q, K, V) = \text{Softmax}\left( \frac{Q \cdot K^T}{\sqrt{d_k}} \right) \cdot V
$$

除以 $\sqrt{d_k}$对 $Q \cdot K^T$的值进行了缩放，使得其值域更合理。从数学期望和方差的角度来看，这种缩放可以让 Softmax 函数的输入值更加稳定，避免值过大导致的梯度消失问题

#### 3.2.2 注意力计算

```python
def attention(query,key,value,mask = None,dropout = None):
    """    
Args:
	d_model = d_k * num_heads
    query: 查询张量，形状为 [batch_size, num_heads, seq_len_q, d_k]
    key: 键张量，形状为 [batch_size, num_heads, seq_len_k, d_k] 通常seq_len_k = seq_len_v
    value: 值张量，形状为 [batch_size, num_heads, seq_len_v, d_v]，
    mask: 掩码张量，形状为 [batch_size, 1, seq_len_q, seq_len_k] 或广播兼容形状
    dropout: Dropout层实例，用于正则化       
Returns:
    output: 注意力输出，形状为 [batch_size, num_heads, seq_len_q, d_v]
    p_attn: 注意力权重分布，形状为 [batch_size, num_heads, seq_len_q, seq_len_k]
    """
    # 获取最后一个维度的大小（特征维度）
    d_k = query.size(-1)
    
    # 计算注意力分数并进行缩放
    scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)
    
    # 应用掩码（将需要屏蔽的位置设为负无穷）
    if mask is not None:
        # 确保掩码与分数张量形状兼容，并处理不同掩码格式
        scores = scores.masked_fill(mask, -torch.inf)
    
    # 计算注意力权重分布
    p_attn = F.softmax(scores, dim=-1)
    
    # 应用dropout
    if dropout is not None:
        p_attn = dropout(p_attn)
    
    # 计算最终输出
    output = torch.matmul(p_attn, value)
    
    return output, p_attn

```

#### 3.2.3 多头注意力机制

```properties
多头注意力的核心思想是将注意力机制分成多个并行的 “头”，每个头独立计算注意力，让模型去关注不同方面的信息,最后拼接结果,实验表明可以提升模型效果。
```

```python
def clones(module, N):
    return nn.ModuleList([copy.deepcopy(module) for _ in range(N)])

class MultiHeadedAttention(nn.Module):
    def __init__(self, head: int, embedding_dim: int, dropout: float = 0.1):
        super(MultiHeadedAttention, self).__init__()
        # 确保嵌入维度能被头数整除
        assert embedding_dim % head == 0, "嵌入维度必须能被头数整除"
        
        # 每个头的维度
        self.d_k = embedding_dim // head
        self.head = head
        self.embedding_dim = embedding_dim
        
        # 4个线性层：3个用于Q、K、V的线性变换，1个用于最终输出
        self.linears = clones(nn.Linear(embedding_dim, embedding_dim), 4)
        
        # 存储注意力权重，便于后续分析
        self.attn = None
        
        # Dropout层用于正则化
        self.dropout = nn.Dropout(p=dropout)

    def forward(self, query, key, value, mask = None):
        batch_size = query.size(0)
        
        # 处理掩码：增加头维度以便广播
        if mask is not None:
            # [batch_size, 1, seq_len_q, seq_len_k]
            mask = mask.unsqueeze(1)
        
        # 1. 通过线性层并分割成多个头
        # 形状变化: [batch_size, seq_len, embedding_dim] 
        # -> [batch_size, seq_len, head, d_k] -> [batch_size, head, seq_len, d_k]
        query, key, value = [
            linear(x).view(batch_size, -1, self.head, self.d_k).transpose(1, 2)
            for linear, x in zip(self.linears[:3], (query, key, value))
        ]
        
        # 2. 计算缩放点积注意力
        x, self.attn = attention(query, key, value, mask=mask, dropout=self.dropout)
        
        # 3. 拼接所有头的结果
        # 形状变化: [batch_size, head, seq_len_q, d_k] 
        #[batch_size, seq_len_q, head, d_k] -> [batch_size, seq_len_q, embedding_dim]
        x = x.transpose(1, 2).contiguous().view(batch_size, -1, self.head * self.d_k)
        
        # 4. 应用最终的线性层
        return self.linears[-1](x)

```

```python
# 两个全连接层，增强模型的拟合能力
class PositionwiseFeedForward(nn.Module):
    def __init__(self, d_model, d_ff, dropout = 0.1):
        super().__init__()
        self.w1 = nn.Linear(d_model, d_ff)  # 第一个线性层：升维
        self.w2 = nn.Linear(d_ff, d_model)  # 第二个线性层：降维，d_ff 通常为d_model的4倍
        self.dropout = nn.Dropout(p=dropout)

    def forward(self, x: Tensor) -> Tensor:
        # 先通过第一个线性层升维，再经过ReLU激活和Dropout，最后通过第二个线性层降维
        x = self.w1(x)          # [batch_size, seq_len, d_ff]
        x = F.relu(x)           # 应用ReLU激活函数
        x = self.dropout(x)     # 应用Dropout
        x = self.w2(x)          # [batch_size, seq_len, d_model]
        return x
```

### 3.4 规范化层

```python
# 随着网络深度的增加，模型参数会出现过大或过小的情况，进而可能影响模型的收敛，因此进行规范化，将参数规范到某个特征范围内，辅助模型快速收敛。
# BatchNorm：针对同一个特征不同的样本进行规范化--->CV
# LayerNorm：针对一个样本，不同的特征进行规范化--->NLP

class LayerNorm(nn.Module):
    """
    层归一化（Layer Normalization）
    对每个样本的特征维度进行归一化，适用于序列数据
    公式：y = a * (x - μ) / (√σ² + ε) + b
    其中μ是均值，σ²是方差，a和b是可学习的缩放和平移参数
    """
    def __init__(self, features, eps = 1e-6):
        super().__init__()
        # 可学习的缩放参数（初始化为1）
        self.scale = nn.Parameter(torch.ones(features))
        # 可学习的平移参数（初始化为0）
        self.shift = nn.Parameter(torch.zeros(features))
        self.eps = eps

    def forward(self, x: Tensor) -> Tensor:
        # 计算最后一个维度的均值（保持维度以便广播）
        mean = x.mean(dim=-1, keepdim=True)  # 形状: [batch_size, ..., 1]
        # 计算最后一个维度的标准差（保持维度以便广播）
        std = x.std(dim=-1, keepdim=True, unbiased=False)  # 使用总体标准差
        # 应用层归一化公式
        normalized = (x - mean) / (std + self.eps)
        # 应用缩放和平移
        output = self.scale * normalized + self.shift
        return output
    
```



### 3.5 子层连接结构

```python
# # 子层连接结构 子层(前馈全连接层 或者 注意力机制层)+ norm层 + 残差连接
class SublayerConnection(nn.Module):
    def __init__(self, size: int, dropout: float = 0.1):
        super().__init__()
        self.norm = LayerNorm(size)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, sublayer):
        # 先归一化，再通过子层，应用dropout，最后与原始输入残差连接
        # 这种预归一化（Pre-LayerNorm）方式在现代Transformer实现中更常见
		# sublayer 函数入口地址 子层函数(前馈全连接层 或者 注意力机制层函数的入口地址)
        return x + self.dropout(sublayer(self.norm(x)))
   
```

```python
# 每个编码器层完成一次对输入的特征提取过程, 即编码过程.


class EncoderLayer(nn.Module):
    """
    Transformer编码器层
    由两个子层组成：多头自注意力层 + 前馈神经网络层
    每个子层都包含残差连接和层归一化
    """
    def __init__(self,size,self_attn,feed_forward,dropout):
        """
        初始化编码器层
        Args:
            size: 输入特征维度（通常为d_model）
            self_attn: 自注意力子层函数
            feed_forward: 前馈神经网络子层函数
            dropout: Dropout概率
        """
        super().__init__()
        self.self_attn = self_attn
        self.feed_forward = feed_forward
        # 创建两个子层连接（残差连接+层归一化）
        self.sublayers = clones(SublayerConnection(size, dropout), 2)
        self.size = size

    def forward(self, x, mask):
        """        
Args:
x: 输入张量，形状为 [batch_size, seq_len, size]
mask: 掩码张量，用于屏蔽填充位置，形状为 [batch_size, 1, 1, seq_len]，在多头注意力计算中，注意力分数矩阵（scores）的形状为：[batch_size, num_heads, seq_len_q, seq_len_k]，当 num_heads=8，seq_len_q=5 时，形状 [batch_size, 1, 1, seq_len] 会自动广播为 [batch_size, 8, 5, seq_len]，与注意力分数矩阵形状完全匹配。填充掩码的原始形状通常是 [batch_size, seq_len]（每个样本标记哪些位置是 <PAD>），通过两次扩展维度（unsqueeze(1).unsqueeze(1)）得到 [batch_size, 1, 1, seq_len]
            
        Returns:
            编码后的张量，形状与输入相同
        """
        # 第一个子层：自注意力 + 残差连接
        x = self.sublayers[0](x,lambda x_val: self.self_attn(x_val,x_val, x_val, mask))
        # 第二个子层：前馈网络 + 残差连接
        x = self.sublayers[1](x, self.feed_forward)
        
        return x
    
```



### 3.7 编码器

```python
class Encoder(nn.Module):
    """
    Transformer编码器
    由N个相同的编码器层堆叠而成，最后通过一层归一化输出
    作用：将输入序列编码为包含上下文信息的特征表示
    """
    def __init__(self, layer, num_layers):
        super().__init__()
        # 克隆N个编码器层
        self.layers = clones(layer, num_layers)
        # 最终的层归一化
        self.norm = LayerNorm(layer.size)

    def forward(self, x, mask):
        """
        Args:
            x: 输入张量，形状为 [batch_size, seq_len, d_model]
            mask: 填充掩码张量，形状为 [batch_size, 1, 1, seq_len]
            
        Returns:
            编码后的特征张量，形状为 [batch_size, seq_len, d_model]
        """
        # 依次通过所有编码器层
        for layer in self.layers:
            x = layer(x, mask)
        # 最后应用层归一化
        return self.norm(x)
    
```



## 4 解码器部分

### 4.1 解码器层

```python
class DecoderLayer(nn.Module):
    """
    Transformer解码器层
    包含三个子层：
    1. 解码器自注意力层（带序列掩码，防止未来信息泄露）
    2. 编码器-解码器交叉注意力层（关注编码器输出）
    3. 位置前馈神经网络层
    每个子层均配备残差连接和层归一化
    """
    def __init__(self,size,self_attn,src_attn,feed_forward,dropout):
        """
        Args:
            size: 特征维度（通常等于d_model）
            self_attn: 解码器自注意力机制（Q=K=V）
            src_attn: 编码器-解码器交叉注意力机制（Q来自解码器，K=V来自编码器）
            feed_forward: 位置前馈神经网络
            dropout: Dropout概率
        """
        super().__init__()
        self.size = size
        self.self_attn = self_attn
        self.src_attn = src_attn
        self.feed_forward = feed_forward
        # 创建3个子层连接模块（残差连接+层归一化）
        self.sublayers = clones(SublayerConnection(size, dropout), 3)

    def forward(self,x,memory,source_mask,target_mask):
        """
Args:
x: 解码器输入张量，形状为 [batch_size, tgt_seq_len, size]
memory: 编码器输出张量（上下文信息），形状为 [batch_size, src_seq_len, size]
source_mask: 源序列掩码，用于屏蔽源序列中的填充位置，形状为 [batch_size, 1, 1, src_seq_len]
target_mask: 目标序列掩码，用于屏蔽目标序列中的填充位置和未来信息，形状为 [batch_size, 1, tgt_seq_len, tgt_seq_len]，target_mask的形状 [batch_size, 1, tgt_seq_len, tgt_seq_len] 能通过广播自动适配注意力分数矩阵的形状 [batch_size, num_heads, tgt_seq_len, tgt_seq_len]
        
Returns:
解码后的特征张量，形状为 [batch_size, tgt_seq_len, size]
        """
        # 1. 解码器自注意力子层：使用目标序列掩码防止未来信息泄露
        x = self.sublayers[0](x,lambda x_val: self.self_attn(x_val, x_val, x_val, target_mask))
        
        # 2. 编码器-解码器交叉注意力子层：关注编码器输出的上下文信息
        x = self.sublayers[1](x,lambda x_val: self.src_attn(x_val, memory, memory, source_mask))
        
        # 3. 前馈神经网络子层：进行特征的非线性变换
        x = self.sublayers[2](x, self.feed_forward)
        
        return x
    
```

### 4.2 解码器

```python
class Decoder(nn.Module):

    def __init__(self, layer, N):
        # 参数layer 解码器层对象
        # 参数N 解码器层对象的个数

        super(Decoder, self).__init__()

        # clones N个解码器层
        self.layers = clones(layer, N)

        # 定义规范化层
        self.norm = LayerNorm(layer.size)

    def forward(self, x, memory, source_mask, target_mask):

        # 数据以此经过各个子层
        for layer in self.layers:
            x = layer(x, memory, source_mask, target_mask)

        # 数据最后经过规范化层
        return self.norm(x)
```



## 5 输出部分

```python
class Generator(nn.Module):
    """
    Transformer的生成器模块
    将解码器输出的特征向量映射到词汇表空间，生成最终的预测概率分布
    
    作用：在序列生成任务（如机器翻译）中，将解码器的隐藏状态转换为每个位置的词汇概率
    """
    def __init__(self, d_model: int, vocab_size: int):
        """
        Args:
            d_model: 输入特征维度（与解码器输出维度一致）
            vocab_size: 目标词汇表大小
        """
        super().__init__()
        # 线性层：将d_model维度映射到词汇表维度
        self.project = nn.Linear(d_model, vocab_size)

    def forward(self, x: Tensor) -> Tensor:
        """
        前向传播：将特征映射为词汇概率分布
        Args:
            x: 解码器输出张量，形状为 [batch_size, seq_len, d_model]   
        Returns:
            词汇概率的对数形式，形状为 [batch_size, seq_len, vocab_size]
        """
        # 先通过线性层映射到词汇表维度，再应用log_softmax获取对数概率
        # 使用log_softmax而非softmax，便于后续计算负对数似然损失(NLLLoss)
        return F.log_softmax(self.project(x), dim=-1)
    
```



## 6 Transformer模型搭建

### 6.1 编码器-解码器构建

```python
class EncoderDecoder(nn.Module):
    """
    Transformer的编码器-解码器架构
    整合了完整的序列到序列（Sequence-to-Sequence）处理流程：
    输入序列 → 编码器 → 解码器 → 输出序列概率分布
    """
    def __init__(self,encoder,decoder,src_embed,tgt_embed,generator):
        """
        初始化编码器-解码器架构
        Args:
            encoder: 编码器模块
            decoder: 解码器模块
            src_embed: 源序列嵌入层（词嵌入+位置编码）
            tgt_embed: 目标序列嵌入层（词嵌入+位置编码）
            generator: 输出生成器（将解码器输出映射为词汇概率）
        """
        super().__init__()
        self.encoder = encoder
        self.decoder = decoder
        self.src_embed = src_embed  # 源序列嵌入：词嵌入+位置编码
        self.tgt_embed = tgt_embed  # 目标序列嵌入：词嵌入+位置编码
        self.generator = generator  # 最终输出层

    def forward(self,source,target,source_mask,target_mask):
        """
        前向传播：完整的编码-解码流程
        Args:
            source: 源序列输入，形状为 [batch_size, src_seq_len]
            target: 目标序列输入，形状为 [batch_size, tgt_seq_len]
            source_mask: 源序列掩码，形状为 [batch_size, 1, 1, src_seq_len]
            target_mask: 目标序列掩码，形状为 [batch_size, 1, tgt_seq_len, tgt_seq_len]  
        Returns:
            目标序列的概率分布，形状为 [batch_size, tgt_seq_len, vocab_size]
        """
        # 1. 编码过程：源序列 → 嵌入 → 编码器 → 上下文特征
        memory = self.encode(source, source_mask)
        
        # 2. 解码过程：目标序列 → 嵌入 → 解码器（结合上下文特征）→ 生成概率分布
        return self.decode(memory, source_mask, target, target_mask)

    def encode(self, source, source_mask):
        """
        编码过程：将源序列转换为上下文特征
        Args:
            source: 源序列输入
            source_mask: 源序列掩码  
        Returns:
            编码器输出的上下文特征，形状为 [batch_size, src_seq_len, d_model]
        """
        # 源序列嵌入（词嵌入+位置编码）→ 编码器
        return self.encoder(self.src_embed(source), source_mask)

    def decode(self,memory,source_mask,target,target_mask):
        """
        解码过程：结合上下文特征生成目标序列概率
        Args:
            memory: 编码器输出的上下文特征
            source_mask: 源序列掩码
            target: 目标序列输入
            target_mask: 目标序列掩码  
        Returns:
            目标序列的概率分布
        """
        # 目标序列嵌入（词嵌入+位置编码）→ 解码器 → 生成器
        decoder_output = self.decoder(self.tgt_embed(target), memory, source_mask, target_mask)
        return self.generator(decoder_output)
  
```



### 6.2 Tansformer模型构建

```python
def make_model(source_vocab,target_vocab,N=6,d_model=512,d_ff=2048, head=8,dropout_p=0.1):
    """
    构建完整的Transformer模型
    Args:
        source_vocab: 源语言词汇表大小
        target_vocab: 目标语言词汇表大小
        N: 编码器/解码器层数，默认6
        d_model: 模型维度，默认512
        d_ff: 前馈网络中间层维度，默认2048（512*4）
        head: 注意力头数量，默认8
        dropout_p: Dropout概率，默认0.1  
    Returns:
        完整的EncoderDecoder模型实例
    """
    # 创建深拷贝函数，确保各层参数独立
    copy_fn = copy.deepcopy
    
    # 1. 实例化核心组件
    # 多头注意力机制
    attn = MultiHeadAttention(embedding_dim=d_model, head=head, dropout=dropout_p)
    # 前馈神经网络
    ff = FeedForward(d_model=d_model, d_ff=d_ff, dropout=dropout_p)
    # 位置编码
    position_encoding = PositionEncoding(d_model=d_model, dropout=dropout_p, max_len=2000)
    
    # 2. 构建编码器和解码器
    # 编码器层
    encoder_layer = EncoderLayer(
        size=d_model,
        self_attn=copy_fn(attn),
        feed_forward=copy_fn(ff),
        dropout=dropout_p
    )
    # 解码器层
    decoder_layer = DecoderLayer(
        size=d_model,
        self_attn=copy_fn(attn),
        src_attn=copy_fn(attn),
        feed_forward=copy_fn(ff),
        dropout=dropout_p
    )
    
    # 3. 构建嵌入层（词嵌入+位置编码）
    src_embed = nn.Sequential(
        Embeddings(vocab_size=source_vocab, d_model=d_model),
        copy_fn(position_encoding)
    )
    tgt_embed = nn.Sequential(
        Embeddings(vocab_size=target_vocab, d_model=d_model),
        copy_fn(position_encoding)
    )
    
    # 4. 构建生成器（输出层）
    generator = Generator(d_model=d_model, vocab_size=target_vocab)
    
    # 5. 组装完整模型
    model = EncoderDecoder(
        encoder=Encoder(encoder_layer, N),
        decoder=Decoder(decoder_layer, N),
        src_embed=src_embed,
        tgt_embed=tgt_embed,
        generator=generator
    )
    
    # 6. 参数初始化：使用Xavier均匀分布初始化所有可学习参数
    for p in model.parameters():
        if p.dim() > 1:  # 仅对矩阵参数进行初始化（忽略偏置等1D参数）
            xavier_uniform_(p)
    
    return model
    
```

## 1 迁移学习的概念

```properties
简单来说别人训练好的预训练模型。一般预训练模型具备复杂的网络模型结构,一般是在大量的语料下训练完成的
```

### 1.1 预训练语言模型的类别

```properties
现在我们接触到的预训练语言模型，基本上都是基于transformer这个模型迭代而来的
因此划分模型类别的时候，以transformer架构来划分：
Encoder-Only: 只有编码器部分的模型，代表：BERT
Decoder-Only: 只要解码器部分的模型，代表：GPT
Encoder-Decoder: 本质就transformer架构，代表：T5
```

### 1.2 微调(Fine-tuning)

```properties
定义:一般是对预训练语言模型，进行垂直领域数据的微调，可以将预训练模型的参数全部微调或者部分微调或者不微调，但是一般我们在做任务的时候，会在预训练模型后加入自定义网络，自定义网络模型的参数需要训练
```

### 1.3 迁移学习的两种方式

```properties
开箱即用: 当预训练模型的任务和我们要做的任务相似时，可以直接使用预训练模型来解决对应的任务
微调: 进行垂直领域数据的微调，一般在预训练网络模型后，加入自定义网络，自定义网络模型的参数需要训练，但是预训练模型的参数可以全部微调或者部分微调或者不微调。
```

```python
            label = 0
        else:
            s2 = s2_original
            label = 1

        return s1, s2, label


def collate_fn(batch_data):
    """处理批次数据，生成模型输入"""
    s1_list = [item[0] for item in batch_data]
    s2_list = [item[1] for item in batch_data]
    labels = torch.tensor([item[2] for item in batch_data], dtype=torch.long).to(device)

    # 用更现代的tokenizer接口处理句子对
    inputs = tokenizer(
        s1_list, s2_list,  # 分别传入s1和s2，自动处理句子分隔
        truncation=True,
        padding='max_length',
        max_length=args.max_len,
        return_tensors='pt'
    )

    return inputs['input_ids'], inputs['attention_mask'], inputs['token_type_ids'], labels


class NSPModel(nn.Module):
    def __init__(self):
        super(NSPModel, self).__init__()
        self.bert = bert_model
        self.fc = nn.Linear(768, 2)  # 二分类（0/1）
        # 初始化线性层权重
        nn.init.xavier_uniform_(self.fc.weight)

    def forward(self, input_ids, attention_mask, token_type_ids):
        # 根据是否冻结BERT决定是否计算梯度
        with torch.no_grad() if args.freeze_bert else torch.enable_grad():
            bert_output = self.bert(
                input_ids=input_ids,
                attention_mask=attention_mask,
                token_type_ids=token_type_ids
            )
        # 使用[CLS]位置的输出做预测（BERT预训练中NSP任务的标准做法）
        cls_output = bert_output.pooler_output
        logits = self.fc(cls_output)
        return logits


def train():
    # 加载数据
    train_dataset = NSPDataset(args.train_data)
    train_dataloader = DataLoader(
        train_dataset,
        batch_size=args.batch_size,
        shuffle=True,
        collate_fn=collate_fn,
        drop_last=False
    )
    # 加载验证集
    val_dataset = NSPDataset(args.test_data)
    val_dataloader = DataLoader(
        val_dataset,
        batch_size=args.batch_size,
        shuffle=False,
        collate_fn=collate_fn
    )

    # 初始化模型
    model = NSPModel().to(device)
    logger.info("模型初始化完成")

    # 定义损失函数和优化器
    criterion = nn.CrossEntropyLoss()
    # 仅优化需要更新的参数（若冻结BERT，则只优化fc层）
    optimizer = torch.optim.AdamW(
        filter(lambda p: p.requires_grad, model.parameters()),
        lr=args.lr,
        eps=1e-8
    )

    # 学习率调度器（提升训练稳定性）
    total_steps = len(train_dataloader) * args.epochs
    scheduler = get_linear_schedule_with_warmup(
        optimizer,
        num_warmup_steps=0,
        num_training_steps=total_steps
    )

    best_val_acc = 0.0  # 记录最佳验证准确率

    for epoch in range(args.epochs):
        start_time = time.time()
        epoch_loss = 0.0
        model.train()

        logger.info(f"\n===== 第 {epoch+1}/{args.epochs} 轮训练 =====")
        for batch_idx, (input_ids, attention_mask, token_type_ids, labels) in enumerate(tqdm(train_dataloader)):
            # 数据移至设备
            input_ids = input_ids.to(device)
            attention_mask = attention_mask.to(device)
            token_type_ids = token_type_ids.to(device)
            labels = labels.to(device)

            # 前向传播
            outputs = model(input_ids, attention_mask, token_type_ids)
            loss = criterion(outputs, labels)

            # 反向传播与优化
            optimizer.zero_grad()
            loss.backward()
            torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)  # 梯度裁剪
            optimizer.step()
            scheduler.step()

            epoch_loss += loss.item()

            # 每10个批次打印一次中间结果
            if (batch_idx + 1) % 10 == 0:
                logger.info(f"批次 {batch_idx+1}/{len(train_dataloader)}，平均损失: {epoch_loss/(batch_idx+1):.4f}")

        # 验证集评估
        val_acc = evaluate(model, val_dataloader)
        avg_loss = epoch_loss / len(train_dataloader)

        # 保存最佳模型
        if val_acc > best_val_acc:
            best_val_acc = val_acc
            model_path = f"{args.output_dir}/nsp_best.pt"
            torch.save(model.state_dict(), model_path)
            logger.info(f"保存最佳模型至 {model_path}，验证准确率: {val_acc:.4f}")

        # 打印轮次总结
        logger.info(f"轮次 {epoch+1} 结束:")
        logger.info(f"平均损失: {avg_loss:.4f}，验证准确率: {val_acc:.4f}")
        logger.info(f"耗时: {time.time() - start_time:.2f} 秒")

    return best_val_acc


def evaluate(model, dataloader):
    """评估模型在数据集上的准确率"""
    model.eval()
    correct = 0
    total = 0

    with torch.no_grad():
        for input_ids, attention_mask, token_type_ids, labels in tqdm(dataloader, desc="评估中"):
            input_ids = input_ids.to(device)
            attention_mask = attention_mask.to(device)
            token_type_ids = token_type_ids.to(device)
            labels = labels.to(device)

            outputs = model(input_ids, attention_mask, token_type_ids)
            preds = torch.argmax(outputs, dim=1)

            correct += (preds == labels).sum().item()
            total += labels.size(0)  # 总样本数（正确计算方式）

    model.train()  # 恢复训练模式
    return correct / total if total > 0 else 0.0


def test():
    """测试最佳模型性能"""
    test_dataset = NSPDataset(args.test_data)
    test_dataloader = DataLoader(
        test_dataset,
        batch_size=args.batch_size,
        shuffle=False,
        collate_fn=collate_fn
    )

    # 加载最佳模型
    model = NSPModel().to(device)
    model_path = f"{args.output_dir}/nsp_best.pt"
    model.load_state_dict(torch.load(model_path, map_location=device))
    logger.info(f"已加载最佳模型: {model_path}")

    # 评估测试集
    test_acc = evaluate(model, test_dataloader)
    logger.info(f"测试集准确率: {test_acc:.4f}")
    return test_acc


if __name__ == '__main__':
    best_val_acc = train()
    logger.info(f"训练完成，最佳验证准确率: {best_val_acc:.4f}")
    test_acc = test()
    logger.info(f"测试完成，最终准确率: {test_acc:.4f}")
```



## 4 BERT系列模型

### 4.1 BERT模型

BERT（Bidirectional Encoder Representations from Transformers）是基于 Transformer 架构的预训练语言模型，其核心结构采用**双向 Transformer 编码器**，整体设计强调对上下文的双向理解。

**1. 整体架构**

BERT 的结构可分为三部分：输入层、编码器层、输出层

**2. 输入层（Input Embedding）**

**Token Embedding**：将每个分词（Token）映射到固定维度的向量（如 768 维），包含一个特殊的`[CLS]`标记（用于分类任务）和`[SEP]`标记（用于分隔句子）。

**Segment Embedding**：区分句子对（如 “句子 A” 和 “句子 B”），用 0/1 表示不同句子。

**Position Embedding**：编码 Token 在序列中的位置信息（BERT 使用可学习的位置嵌入，而非 Transformer 原有的正弦余弦函数）。
每个 Token 的最终嵌入 = Token Embedding + Segment Embedding + Position Embedding。

**3. Transformer 编码器层**

BERT 的主体是多个**Transformer 编码器**的堆叠（数量因模型大小而异）：**BERT-base**：12 层编码器，隐藏层维度 768，12 个注意力头，**BERT-large**：24 层编码器，隐藏层维度 1024，16 个注意力头

**4. BERT 模型的预训练任务**

预训练任务是指在大规模无标注文本数据上，**为模型设计特定的学习目标，让模型通过学习这些任务自动捕捉语言规律，从而获得通用的语言理解能力。**简单来说，预训练任务就像给模型 “布置作业”，让它在海量文本中 “自学” 语言知识，为后续处理具体任务（如分类、翻译等）打下基础。

> **Masked Language Model（MLM，掩码语言模型）**

**任务设计**：随机选择输入文本中 **15% 的 token** 进行特殊处理：**80% 的概率**用`[MASK]`替换该 token（如 “我爱自然语言处理”→“我爱 [MASK] 语言处理”）；**10% 的概率**用随机词汇替换该 token（如 “我爱自然语言处理”→“我爱苹果语言处理”）；**10% 的概率**保持原 token 不变（干扰模型，避免过度依赖`[MASK]`标记）。

**学习目标**：让模型根据上下文预测被处理的 token 的原始值。迫使模型学习双向上下文信息

> **Next Sentence Prediction（NSP，下一句预测）**

**任务设计**：将文本对（句子 A 和句子 B）作为输入，其中 50% 的概率 B 是 A 的真实下一句，50% 的概率 B 是从语料中随机抽取的无关句子。

**学习目标**：让模型判断句子 B 是否是句子 A 的下一句，帮助模型学习句子之间的逻辑关系（如因果、递进等），增强对篇章级语义的理解。
